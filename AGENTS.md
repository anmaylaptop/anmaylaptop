# Frontend

- This project was bootstrapped by Lovable. Therefore, it must compatible with Supbase
- Responsive UI is mandatory

# Documentation

- All documentation files must be placed in `docs/`

# Development rules

- Never run dev server to check your code, ask user to do that if you need logs or information.
- Never run supabase commands to change database, ask user to do that manually.
- Use `eslint` and `tsc` to check for typing issues and validate your code.
